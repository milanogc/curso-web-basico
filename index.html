Tags

  • O navegador enxerga as tags como sinalizadores que indicam o modo como deve ser exibido o conteúdo por elas delimitado
  • Os nomes das tags possuem em seu entorno os sinais de menor (<) e maior (>) e a tag de fechamento (que indica o fim do conteúdo) tem o nome antecedido por uma barra (/)
  • É recomendável o uso de letras minúsculas para os nomes das tags
<tag> ... conteúdo da tag ... </tag>

Tags x elementos

As tags marcam o início e o fim de um elemento. Portanto, um elemento é tudo o que é compreendido pela tag de início e de fim, e pode, inclusive, conter outros elementos.

Por exemplo, o elemento body é tudo aquilo que está entre as tags <body> e </body>.

Tabela, linhas e células

Tabelas são definidas com a tag table, que são divididas em linhas com a tag tr e estas são dividas com a tag td (ou th).

  • Célula de cabeçalho: criada com a tag th

  • Célula de dados: criada com a tag td
    • Normalmente o conteúdo não apresenta destaques e está alinhado à esquerda

Grupos de linhas

Os grupos de linhas servem para agrupar porções da tabela que tenham a mesma finalidade.

Por exemplo, a divisão entre thead, tbody e tfoot permite que, em tabelas longas, cabeçalho e rodapé sejam repetidos em cada página impressa.

Essas tags são opcionais, mas tornam o documento HTML mais inteligente.

URL

  • Abreviação para Uniform Resource Locator
    • Localizador Padrão de Recursos
  • Define as regras de como se referir aos documentos

Partes de uma URL

protocolo://servidor:porta/c/a/m/i/n/h/o?query_string#fragmento
  • Protocolo: http, https, ftp, ...
  • Servidor: domínio/nome ou endereço IP
  • Porta em que o processo no servidor está executando
    • Se omitida, a porta padrão é usada (http: 80, https: 443)
  • Caminho de acesso ao recurso: permite ao servidor conhecer o lugar onde o recurso está situado
  • Query string: presença indicada pela ? após o caminho
  • Fragmento: presença indicada por # (sempre fica por último na URL)
    • Não é enviado ao servidor

URLs relativas e absolutas

  • URLs relativas: se baseiam no diretório atual, podem apontar para diretórios ancestrais ("../../").
  • URLs absolutas

Atributo target da tag a

A página de um link normalmente é aberta na janela corrente do navegador. Para abrir em outra janela devemos utilizar o atributo target. Quando este atributo é omitido seu valor padrão é _self, o valor _blank determina a abertura do link em outra janela/aba.


  <a href="http://www.google.com" target="_blank">Link para o Google</a>

Formulários HTML

  • Formulários HTML permitem o envio de dados do navegador para o servidor Web
  • Um formulário HTML é feito de um ou mais campos de entrada. Tais campos podem ser de texto (linha única ou de várias linhas), caixas de seleção, botões, checkboxes ou radio buttons

Forms

Para definirmos um formulário HTML utilizamos a tag form.


<form></form>

Inputs

Tag para entrada de dados.


<input>

Atributo type do elemento input

O atributo type especifica o tipo do elemento input a ser mostrado. O tipo padrão é text.

Apesar de não ser um atributo obrigatório do elemento input, é boa prática que sempre seja especificado.

<input type="text">

Atributo name

Todas as tags de entrada de dados precisam de um nome ou não serão enviadas ao servidor. Por isso adicionamos o atributo name à tag de abertura.


<input type="text" name="q">

Query string

  • A query string é composta por uma série de pares campo-valor
  • Dentro de cada par o campo e o valor são separados pelo sinal de igualdade, =
  • Um par é separado do outro por um e comercial, &

nome1=valor1&nome2=valor2&nome3=valor3

Botão de envio

Por padrão, um formulário HTML envia os dados para si mesmo. Para enviar os dados para outro lugar utilizamos o atributo action. O valor do atributo action será a URL para onde queremos enviar os dados.

GET x POST

GET

  • Adequado para consultas
  • A URL pode ser "favoritada"
  • Quantidades de dados limitada
  • Dados menos seguros, pois estão expostos na URL

POST

  • Adequado para escritas
  • Não pode ser "favoritado"
  • Permite grandes quantidades de dados
  • Dados mais seguros, pois estão "escondidos" no corpo da requisição HTTP

Questão

O que é apropriado para requisições GET?

  • Obter um documento
  • Atualizar dados
  • Requisições que podem ser cacheadas
  • Enviar muitos dados ao servidor

Atributo id

O atributo id é um atributo global, pois pode ser utilizado em qualquer elemento HTML. É usado para definir um identificador único ao elemento em todo o documento que, assim, pode ser referenciado facilmente.


<input type="text" name="q" id="q">
O atributo name não é único, pois vários elementos podem ter o mesmo nome, e nem é global, pois somente alguns elementos possuem este atributo.

Rótulos

  • A tag label define um rótulo para um elemento de formulário
  • Provê melhor usabilidade, pois, ao clicar no rótulo, o elemento de formulário associado é selecionado

<!doctype html>
<html lang="pt">

      <label for="q">Buscar por</label>
      <input type="text" name="q" id="q">
      <input type="submit">
    </form>
  </body>
</html>

Radio buttons

Área de texto (textarea)

Um textarea define uma área de texto. Diferentemente do input do tipo text, um textarea pode conter várias linhas.

O que é JavaScript?

JavaScript é uma linguagem de programação que foi originalmente implementada como parte dos navegadores Web para que scripts pudessem ser executados do lado do cliente e interagissem com o usuário.

Ambiente de execução

Atualmente, os navegadores não são as únicas plataformas em que o JavaScript é utilizado. Alguns bancos de dados usam JavaScript como linguagem de configuração e de consulta. O projeto Node.js provê um ambiente de execução JavaScript no servidor. E, até mesmo o SITRAM (SEFAZ-CE), utiliza JavaScript (no servidor) para a definição de regras para o cálculo do ICMS.

Tipos

JavaScript é uma liguagem fracamente tipada (ou dinâmica). Não é necessário declarar o tipo da variável. Os tipos disponíveis são: undefined, string, number, boolean, null e object. O operador typeof verifica o valor e informa o nome do seu tipo.


var a;
typeof a; // "undefined"

a = undefined;
typeof a; // "undefined"

a = "olá, mundo!";
typeof a; // "string"

a = 'olá, mundo!';
typeof a; // "string"
            

a = 42;
typeof a; // "number"

a = true;
typeof a; // "boolean"

a = null;
typeof a; // "object" bug!

a = { b: "c" };
typeof a; // "object"

Arrays

Um array é um objeto especializado para armazenar valores (de qualquer tipo). Acessamos os valores contidos em um array através de índices numéricos começados por 0 (zero).


var produtos = ["arroz", "feijão", "macarrão"]; // array com 3 elementos

console.log(produtos[0]); // arroz (1o elemento)
console.log(produtos[1]); // feijão (2o elemento)
console.log(produtos[2]); // macarrão (3o elemento)
console.log(produtos.length); // 3 (quantidade de elementos)

Objetos

Um objeto é uma coleção de valores (de qualquer tipo) nomeados (pares nome-valor). Acessamos os valores contidos em um objeto através de índices nomeados.


var maca = {nome: 'maçã', preco: 4, unidade: 'kg'};
var banana = {nome: 'banana', preco: 2, unidade: 'kg'};

console.log(maca.nome, maca['nome']); // maçã maçã
console.log(maca.preco, maca['preco']); // 4 4
console.log(maca.unidade, maca['unidade']); // kg kg

console.log(banana); // Object { nome: "banana", preco: 2, unidade: "kg" }

Igualdade e desigualdade estrita

Ao utilizar os operadores de igualdade (==) e desigualde (!=) o JavaScript tenta fazer conversão dos tipos para se tornarem compatíveis.

=== igualdade de valor e tipo (não há conversão)
!== desigualdade de valor e tipo (não há conversão) 

'' == '0'           // false
0 == ''             // true
0 == '0'            // true

false == 'false'    // false
false == '0'        // true
            

false == undefined  // false
false == null       // false
null == undefined   // true

' \t\r\n ' == 0     // true

DOM

Quando uma página HTML é carregada, o navegador cria uma representação da estrutura do documento como uma árvore de objetos. Com esta representação, o JavaScript tem o poder de modificar o documento. Quando a estrutura é modificada, a página na tela reflete as mudanças feitas.

A API DOM (Document Object Model) padroniza como essa árvore de objetos pode ser acessada e manipulada.

Variável document

A variável global document (window.document) dá acesso a árvore de objetos DOM.

Alguns métodos de document:
Método Descrição
getElementById Encontra um elemento pelo id
getElementsByTagName Encontra elementos pelo nome da tag
getElementsByClassName Encontra elementos pelo nome da classe

Vamos testar estes métodos no navegador?


document.getElementById('id');
document.getElementsByTagName('tag');
document.getElementsByClassName('classe');

jQuery

jQuery é uma biblioteca JavaScript que simplifica a interação com o HTML. É usada por cerca de 65% dos 10 milhões de sites mais visitados do mundo (2015).

Por que jQuery?

A sintaxe do jQuery foi desenvolvida para tornar mais simples a "navegação" no documento HTML, selecionar elementos DOM, criar animações, manipular eventos e desenvolver aplicações AJAX. E tudo isso com grande compatibilidade entre os diferentes navegadores.

Seletores jQuery

Os seletores no jQuery são baseados nos seletores CSS e são usados para encontrar (selecionar) elementos baseados em id, classes, atributos, valores de atributos, etc.


document.getElementById('id'); // js puro
$('#id'); // jquery

document.getElementsByTagName('tag'); // js puro
$('tag'); // jquery

document.getElementsByClassName('classe'); // js puro
$('.classe'); // jquery
https://api.jquery.com/category/selectors/

jQuery é poderoso

Evento ready

Uma página não pode ser manipulada com segurança até que o documento esteja pronto. A função $(document).ready espera até que o HTML da página carregue antes de executar a função recebida como parâmetro.



// utilizando o evento ready:

$(document).ready(function() {
   // colocar o código de manipulação do HTML aqui
   console.log("pronto!");
}); // fim do ready

// ou de forma mais concisa:

$(function() {
  // colocar o código de manipulação do HTML aqui
  console.log("pronto!");
}); // fim do ready (implícito)

Ações

A sintaxe do jQuery é orientada pela seleção de elementos HTML e a execução de ações nestes elementos.

Método .css

Obtém ou define um valor de propriedade de um estilo.


// lendo uma propriedade
var cor_de_fundo = $("p").css("background-color");

// definindo uma propriedade
$("p").css("background-color", "#f00");

// definindo mais de uma propriedade ao mesmo tempo
$("p").css({"color": "#000", "background-color": "#f00"});

Outros métodos para trabalhar com estilos: .addClass, .removeClass, .toggleClass.

Métodos .hide e .show

O método .hide esconde os elementos selecionados, e o método .show mostra os elementos selecionados.


$("p").hide(); // "slow", "fast"
$("p").show();
$("p").css("display", "none");

Método .prop

Obtém ou define um valor de uma propriedade.


var marcado = $("#checkbox").prop("checked"); // lê
$("#checkbox").prop("checked", true); // escreve (marca o checkbox)

Método .val

Obtém ou define um valor em elementos de formulário.


var texto = $("input:text").val(); // lê
$("input:text").val("novo texto"); // escreve

Principais eventos do jQuery

https://api.jquery.com/category/events/

Método .on

Este método anexa uma função manipuladora de eventos nos elementos selecionados.


$("p").on("click", function() {

  console.log("Você alterou o texto!");
});

$("input:text").trigger("change"); // dispara o evento programáticamente

Um arquivo WAR segue o formato JAR para empacotar uma aplicação web Java. Um arquivo JAR, por sua vez, é um arquivo compactado no formato ZIP.

Maven

O Maven é uma ferramenta de automação da construção de software. Basicamente, o Maven faz a compilação do código fonte, a gerencia das dependências da aplicação (bibliotecas jars) e dispara a execução de testes automatizados através da configuração realizada no arquivo pom.xml.

Estrutura de diretórios

O Maven cria um projeto esqueleto.

  • src/main/java/: código fonte Java
  • src/main/webapp/: arquivos públicos, podem ser acessados diretamente (html, jsps, imagens, js, css)
  • src/main/webapp/WEB-INF/: arquivos privados, acessíveis via Servlets (jsps, jars, configurações)
  • src/main/webapp/WEB-INF/web.xml: deployment descriptor

Filtros

Um filtro é uma classe Java que implementa a interface javax.servlet.Filter. Filtros são comumente usados para autenticação e auditoria.

Ao implementar a interface Filter, temos que implementar 3 métodos: init, destroy e doFilter.

http://docs.oracle.com/javaee/6/api/javax/servlet/Filter.html

Implementando um filtro


@WebFilter("/*")
public class MeuFiltro implements Filter {
  public void doFilter(ServletRequest request,
    ServletResponse response, FilterChain chain) {
    // ...
    chain.doFilter(request, response);
    // ...
  }
}

Qualquer código colocado antes da chamada chain.doFilter(request, response) será executado antes de chegar no servlet, qualquer código depois será executado depois do servlet.

Codificação UTF-8 em todos os servlets

Para defindir a codificação UTF-8 em todos os sevlets utilizamos um filtro implementado pelo Tomcat.


<filter>
  <filter-name>SetCharacterEncodingFilter</filter-name>
  <filter-class>org.apache.catalina.filters.SetCharacterEncodingFilter</filter-class>
  <init-param>
    <param-name>encoding</param-name>
    <param-value>UTF-8</param-value>
  </init-param>
</filter>
<filter-mapping>
  <filter-name>SetCharacterEncodingFilter</filter-name>
  <url-pattern>/*</url-pattern>
</filter-mapping>
